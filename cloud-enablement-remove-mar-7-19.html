Why SD-WAN?

  • SD-WAN has revolutionized the IT networking space for enterprises by optimizing wide area networks (WAN).
  • SD-WAN is the fabric used for transferring the application traffic through the most optimal route, which is completely scalable.
  • SD-WAN uses encrypted internet tunnels for transfers which guarantee data integrity. Thus, you can connect multiple branch offices on one network without compromising on agility and flexibility

Cisco Intelligent Automation for Networks (CIAN)

Intelligent Network Automation

First Data had 10000+ devices (Catalyst 6500, ISR etc...) in their global network which included Cisco, Nokia and Avaya. CA Spectrum was used as the primary event manager and received approx. 3 Million+ events per day and 10 thousand alarms which were then handled by Network Operation Center (NOC) on a daily basis. Also approximately 100 Incident reports were generated to CA ServiceDesk system. Addressing each alarm with such high volumes in extremely short amount of time was overwhelming for NOC team.




Netpace Solution:

Netpace CIAN solution based on Cisco Process Orchestrator (CPO) played a significant role by automating incident management/troubleshooting tasks which would otherwise had to be addressed manually. CIAN was used as a Virtual TAC in the NOS for the reactive issues including critical alarms such as circuit failure or device down situations.

Another aspect of this implementation was Proactive monitoring and configuration consistency check such as Failover on FWSM, this activity saved time for the design team so that they can focus on more important business related issues. CPO also had to be integrated with the CA Spectrum in order to update the Alarms and also update the CA ServiceDesk ticket.


Following workflows were implemented

  • Troubleshooting Device NOT Responding/Bad Link
  • Detecting Slow User Response from Branch to a Server in Data Center
  • Troubleshooting Latency problem in the network due to spanning tree issues (Add – device detection)
  • Troubleshooting Performance Issue Due to High CPU load on routers and switches
  • HSRP Redundancy pairs configuration checks
  • FWSM Failover Check

Technologies/Tools used
  • Cisco Process Orchestrator (Tidal Enterprise Orchestrator)
  • Microsoft SQL Database Server
  • Microsoft IIS Web Server
  • Cisco Catalyst 6500 Switches
  • Cisco 7200 Routers
  • Cisco ISR
  • Cisco FWSM
Network Automation and Fault Management using CIAC
Overview & Challenge
  • US BANK receives millions of network events from 10000+ devices in their network. Netcool raises about 100000+ alarms/month to the NOC, and the NOC has to respond to these alarms quickly.
  • CPO was positioned to integrate with Netcool so that critical alarms that require in depth troubleshooting would be forwarded to CPO for troubleshooting while raising alarms and also service desk ticket simultaneously.
  • US BANK wanted “ticket enrichment” to augment their resources in the Network Operation Center (NOC).
  • Addressing every alarm with such a high volume was overwhelming and time consuming for the NOC team.
Solution
  • Netpace Automation Solution automated incident management/troubleshooting tasks and was used as a Virtual TAC in the NOC for the reactive issues including critical alarms.
  • CPO was used as orchestrator to coordinate change management activity between HPSM and HPNA, and help in generating the End device configuration based on the template along with user input.
  • This project also focused on automating and orchestrating Unified Communication Change and Incident Management process and saved time and valuable resources at US Bank.
  • CPO was integrated with Netcool to update the Alarm and also update the service desk ticket.
Cisco Network Orchestration Advanced Services

Intelligent Automation Network Implementation

Bureau of Prisons network had 2,500+ network devices in their national network which primarily included Cisco catalyst switches and routers. Also their wide area network, complexes/institutional networks used tools such as CiscoWorks, Tandberg camera and Cisco Unified Communication. This project presented several opportunities to perform network automation and extensive strategies/designs were employed.

The Bureau of Prisons NOC was interested in enhancing and maintaining the stability of their network by automating complicated/time consuming tasks and creating automated troubleshooting to aid in identifying issues, when they do arise.

Solution Overview:

Netpace network automation solution based on Cisco Process Orchestrator (CPO) played an important role by automating tedious, repetitive and time consuming tasks which would otherwise had to be addressed manually. CIAN was used as a virtual engineer in the NOC where it scheduled critical tasks, and provided value added troubleshooting for tickets which came up.

Following workflows were implemented:
  • ACL changes automation used for DVR Camera monitoring
  • Detecting Slow response from branch office to server
  • Troubleshooting Spanning Tree & HSRP affinity match
  • Tandberg video conferencing network connectivity check
  • Bulk network switch port deployment
  • Bulk device running configuration export from CiscoWorks
  • VTP mode and platform compliance check
  • Relay SNMP Traps to CiscoWorks and Other devices
Technologies/Tools used
  • Cisco Process Orchestrator (Tidal Enterprise Orchestrator)
  • Microsoft SQL Database Server
  • Microsoft IIS Web Server
  • Cisco CiscoWorks LAN Management Solution
  • Cisco Catalyst 6500 Switches
